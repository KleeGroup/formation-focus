    Histoire

    Il était une fois un gros site web que s'appelait Facebook. Qui dit Facebook, dit webapp plus grosse que la plus grosse webapp de tes cauchemars; et du coup, propension à se retrouver submergé de bugs plus élevés.

    Les ingénieurs front-end de Facebook, confrontés à une codebase de plus en plus bordélique (personne ne voulant toucher certaines parties de celle-ci) ont dû repenser la structure des composants les plus cruciaux.

    En image

    4 concepts

    • les actions: proviennent du serveur ou d'une interaction user;
    • le dispatcher: reçoit les actions et transmet à qui veut, un peu comme un EventEmitter global;
    • les stores: équivalant du model de l'architecture MVC, contiennent les données et réagissent aux actions que le dispatcher leur transmet;
    • les views: s'occupent du rendu des données dans le DOM et lancent des actions lorsque l'utilisateur effectue certaines actions.

    Les 4 concepts en détail

    Ici, nous allons voir en détail les 4 concepts avec et sans les helpers de focus.

    Actions

    ## Action - Simple (action interne) On définit une action avec un __action creator__ ```javascript import { appDispatcher } from "../dispatcher"; import { actionTypes } from "../constants"; // Actions du basket export const basketActions = { // Action creator : addToBasket addToBasket(productId) { // Dispatch l'action au travers du dispatcher appDispatcher.handleViewAction({ // Type de l'action type: actionTypes.ADD_TO_BASKET, // Payload de l'action productId: productId, }); }, }; ```
    ## Action - Complexe (appel server) Ici, l'action va appeler une API sur le backend et dispatcher le résultat ```javascript import { appDispatcher } from "../dispatcher"; import { actionTypes } from "../constants"; import { basketService } from "../services"; // Actions du basket export const basketActions = { // Action creator : orderBasket orderBasket() { // Appel du service basketService.orderBasket().then(orderStatus => { // Dispatch du resultat du service appDispatcher.handleServerAction({ type: actionTypes.ORDER_BASKET, orderStatus: orderStatus, }); }) }, }; ```
    ## Actions avec __actionBuilder__ L'action builder est là pour simplifier la mise à jour d'un store à partir du résultat d'un appel au serveur via un service ```javascript import actionBuilder from "focus-core/application/action-builder"; import basketServices from "../services"; export const basketActions = { // Action qui sera disponible addToBasket: actionBuilder({ // Service qui sera appelé service: basketServices.addToBasket, // Noeud du store qui est update node: "basket", // Statut à la fin de l'action status: "updated" }) }; // => on pourra faire basketActions.addToBasket(productId) ```

    Actions - TL;DR

    • Une action est une fonction
    • Une action peut appeler un service
    • Peut dispatcher au dispatcher: avant, pendant, après l'appel au service
    • On peut utiliser un actionBuilder, il en existe plusieurs :
      • simple: fonctionne avec un coreStore
      • recherche: fonctionne avec un searchStore
      • liste: fonctionne avec un listStore

    Stores

    ## Store Le __store__ s'enregistre auprès du __dispatcher__. Il verra donc toutes les __actions__. C'est au __store__ de savoir s'il doit traiter ou non l'__action__ ```javascript import { appDispatcher } from "../dispatcher"; import { actionTypes } from "../constants"; // Etat initial du store et fonction de mise à jour export const basketStore = createStore({ products: [] }, action => { const { type } = action; switch (action) { case actionTypes.ADD_TO_BASKET: const { productId } = action; // Ajoute le productId à la collection this.products = [...this.products, productId]; break; // [...] Autres actions } }); appDispatcher.register(basketStore.dispatcherCallback); ```
    ## Stores avec __CoreStore__ [CoreStore](https://github.com/KleeGroup/focus-core/tree/master/src/store) est là pour aider à la création de stores multi-noeuds qui seront mis à jour avec un actionBuilder. Chaque noeud aura un status et un état de loading (true ou false) ```javascript import CoreStore from "focus-core/store/CoreStore"; export const basketStore = new CoreStore({ basket }); // => bastketStore aura un noeud nommé "basket" ```

    Stores - TL;DR

    • Un store est un objet qui contient une grappe de données
      • Store simple : CoresSore
      • Stores spécialisés : ListStore, AdvancedSearchStore, QuickSearchStore
    • Un store notifie: d'un changement d'état, d'une erreur
    • Chaque noeud de données a un statut et un état de chargement

    Services

    ## Service ```javascript import basketApis from '../apis'; // Basket service export basketService { orderBasket() { return basketApis.oderBasket(); }, addToBacket(productId) { return basketApis.addToBasket({}, { productId }) } // Api méthode signature : (urlParams, bodyParam, options) } ``` - 80% sont un passe plat vers une API - 20% peuvent réaliser des actions complexes

    Views

    ## View - dispatch d'une action On émet une action lors d'une action utilisateur ```javascript import { React } from "react"; import { basketActions } from "../actions"; import { Button } from "../components"; export const Product = ({ productId }) => ( <div class="product"> {/* Component stuff */} <button> {/* Appel de l'action creator */} onClick={() => basketActions.addToBasket(productId)} label="Ajouter au panier" </button> </div> ); ``` A chaque clic sur le bouton, l'action ADD_TO_BASKET est passée au dispatcher
    ## View - Ecoute d'un store ```javascript import { React } from "react"; import { basketStore } from "../stores"; export const Basket = React.createClass({ // State initial du composant getInitialState() { return { products: basketStore.products }; }, // Fonction appellée au changement de données dans le store _onStoreChange() { this.setState({ products: basketStore.products }); }, // Le composant s'enregistre auprès du store pour recevoir les modifications componentDidMount() { basketStore.addChangeListner(this._onStoreChange); }, // Le composant annule sa souscription auprès du store componentWilltUnmount() { basketStore.removeChangeListener(this._onStoreChange); }, render() { <div class="basket"> {`${this.state.products.length} products`} </div> } }); ```
    ## Une view avec __store mixin__ Le [store mixin](https://github.com/KleeGroup/focus-components/blob/master/src/mixin/store-behaviour.js) va écouter le store et mettre à jour le state avec la valeur du/des noeud(s) ```javascript import { React } from "react"; import { storeBehaviour } from "focus-components/mixin"; import { basketStore } from "../stores"; export const Basket = React.createClass({ // Utilisation du store mixin mixins: [storeBehaviour], // Ecoute du noeud "basket" du basketStore stores: [ { store: basketStore, properties: ["basket"] } ], render() { <div class="basket"> {`${this.state.products.length} products`} </div> } }) ```
    ## Une view avec __store connector__ Le [store connector](https://github.com/KleeGroup/focus-components/blob/master/src/behaviours/store/connect.js) va écouter le store et recalculer les props à donner au composant connecté ```javascript import { React } from "react"; import storeConnector from "focus-components/behaviours/store/connect"; import { basketStore } from "../stores"; const connector = storeConnector( { store: basketStore, properties: ["basket"] }, // Fonction de calcul des props du composant connecté props => ({ ...props, // On transferts les props données par le parent // On pourrait appliquer n'importe quelle transformation basket: basketStore.getBasket() }) ); export const Basket = connector(props => ( <div class="basket"> {`${props.basket.products.length} products`} </div> )); ```

    Store Mixin

    Store Connector
    Applatit tous les noeuds écoutés dans le state Chaque noeud est reçu en props
    S'il y a des conflits de propriétés entre deux noeuds, la dernière va écraser Les noeuds restant séparés, pas de risques de conflits
    Si une clé disparait entre deux updates, on va garder l'ancienne valeur dans le state Pas de problèmes 'bro

    Flux TL;DR

    • L'architecture est unidirectionnelle
    • On raisonne en actions, qui sont déclenchées par la vue ou le serveur
    • Toutes les actions passent par le dispatcher
    • Seuls les stores signalent aux vues qu'il faut se mettre à jour

    Focus2.js

    Architecture d'une app Focus2

    Dossiers d'une app Focus2

    • actions/ : toutes les actions organisées par module
    • components/ : tous les composants communs à plusieurs modules.
    • config/ : la configuration de l'application, domaine, entity, etc.
    • i18n/ : les fichiers de traductions organisés par module
    • initializer/ : le boot de l'application
    • routes/ : les routes organisées par modules
    • services/ : les services organisés par modules
    • stores/ : les stores organisés par modules
    • styles/ : les feuilles de style communes à plusieurs modules
    • views/ : les pages, popins et dialogs organisés par module
    • application.jsx : l'initialisation de la vue principale de l'appli
    • index.js : l'orchestrateur de l'appli

    Les domaines et les définitions

    les domaines

    • Responsables de la validation
    • Du composant de rendu (consultation et/ou édition)
    • Formatage des données
    • On peut les utiliser pour le style
    • Utilisation intelligente avec fieldFor

    example du domaine do-date.js

    ```javascript import moment from "moment"; import InputDate from "focus-components/components/input/date"; export default { // Composant utiliser en édition InputComponent: DateComponent, // Formatteur de données pour de la consultation formatter: date => date ? moment(date, moment.ISO_8601).format('DD/MM/YYYY') : "", // Formats autorisés de saisie => validation format: ['DD/MM/YYYY', 'DD-MM-YYYY', 'D MMM YYYY'] }; ```

    Les définitions

    • Générées depuis le modèle de données
    • Lient le modèle de données aux domaines
    • Chaque objet a des champs
    • Chaque champ a un domaine
    • Chaque champ porte l'information requis / non requis

    example de définitions

    ```javascript export const product = { code: { domain: 'DO_ID', required: true }, title: { domain: 'DO_LABEL_LONG', required: true }, creationDate: { domain: 'DO_DATE', required: false } } ```

    Le form mixin

    Votre pire meilleur amis

    Le form mixin

    • Simplification de la gestion de formulaire
    • Formulaire simples ou complexes (ex: sous-listes)
    • Liaisons de champs entre eux
    • Validation automatique avec le domaine ou custom
    • Personnalisation
    ## Un exemple de composant mis en musique ```javascript import React from "react"; import Panel from "focus-components/components/panel"; import { mixin as formMixin } from "focus-components/common/form"; import { productStore } from "../stores"; import { productionActions } from "../actions"; export const ProductDetailPage = React.createClass({ propTypes: { id: PropTypes.number.isRequired }, // Le mixin mixins: [formMixin], // Ici on donne le path de la definition pour trouver le domaine definitionPath: "product", // Le store qui contient le noeud utilisé par le formulaire stores: [{store: productStore, properties: ["productDetail"]}], // Les actions : doivent s"appeller load & save action: { load: productActions.loadProductDetail, save: productActions.saveProductDetail }, // Ici avoir le formixin oblige a passer par renderContent() renderContent() { return ( // Le panel est capable de recevoir les boutons edit/save/cancel <Panel actions={this._renderActions} title="products.detail.title"> // Affichage des différents champs en consultation ou en édition {this.fieldFor("title")} {this.fieldFor("price")} {this.fieldFor("rate")} {this.fieldFor("description")} </Panel> ); } }); ```

    Le router de l'application

    React Router

    React Router est une librairie qui affiche vos composants en fonction de l'URL de votre navigateur. En gros, c'est indispensable si vous voulez faire une SPA et que vous voulez que l'utilisateur puisse croire que c'est un site normal qu'il peut partager, bookmarker, etc.

    ## React Router - Exemple ```javascript import React from "react"; import { Router, browserHistory } from "react-router"; import { AppLayout } from "../components"; import { ProductDetailPage } from "../views/products/detail"; const productRoutes = [{ path: 'products/:id', component: props => (<ProductDetailPage id={parseInt(props.params.id)} {...props}/>) }]; const routes = { path: `${__BASE_URL__}`, indexRoute: { onEnter: ({ params }, replace) => replace(`${__BASE_URL__}home`) }, component: AppLayout, childRoutes: [...productRoutes, ...errorRoutes] }; export const Application = () => { <Router history={browserHistory} routes={routes} /> } ```
